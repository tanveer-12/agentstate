    agentstatelib — Build Instructions

    Package naming — read this first, refer back always

    Two names. Never mix them up.

    Install from PyPI:

    pip install agentstate-lib

    Import in Python:

    from agentstatelib import SharedState, AgentGraph, StatePatch
from agentstatelib.core.state import SharedState
from agentstatelib.coordination.conflicts import LastWriteWins

    Source folder on disk: agentstatelib/

    In pyproject.toml the project name is agentstate-lib (hyphens). The importable package is agentstatelib (underscores). Every import in every file in this document uses agentstatelib. Every pip install uses agentstate-lib.

    How to read this document

    Every week follows the same structure:

    • Context — why this week matters and what it enables
    • Files to work in — exact paths in the order you write them
    • Detailed instructions per file — imports, classes, methods, fields, docstrings — everything except the actual code characters
    • Docs to write — what user-facing documentation ships with this version
    • Test commands — exact PowerShell commands to run
    • Publish commands — how to ship and deploy docs

    Rules: 1. Run ruff check, mypy --strict, and pytest after every coding session — not once per week, every session 2. Every public class and function gets a docstring — mkdocstrings renders these into your API reference pages 3. Docs ship with every published version — users who install the day you publish must be able to use it immediately 4. Colab is your personal GPU dev tool — it ships nothing to users

    Competitive Positioning

    You build against LangGraph, CrewAI, and AutoGen. Every design decision reinforces one of three things:

    Coordination — typed SharedState, StatePatch pipeline, batch conflict resolution, invariant checking. LangGraph silently merges state with dict reducers. You detect and log every conflict explicitly.

    Observability — append-only event log, OpenTelemetry traces, replay debugger, live dashboard. No existing multi-agent framework ships this as a first-class built-in feature.

    Framework agnosticism — any async callable returning StatePatch is a valid agent. OpenAI, Anthropic, Ollama, Groq, rule-based functions — all identical from your library's perspective. LangGraph is coupled to LangChain.

    Feature                   | agentstatelib | LangGraph | CrewAI | AutoGen
--------------------------|---------------|-----------|--------|--------
Typed shared state        | yes           | dict only | no     | no
Append-only event log     | yes           | no        | no     | no
Conflict detection        | yes           | no        | no     | no
Parallel round execution  | yes           | partial   | no     | no
Replay debugger           | yes           | no        | no     | no
Framework agnostic        | yes           | no        | part   | part
Local model support       | yes           | partial   | part   | part
Built-in observability    | yes           | no        | no     | no

    Local Models: Your Dev Setup vs What Ships to Users

    Your Colab session is your personal GPU access for testing. It ships nothing to users.

    What you use (dev only): - notebooks/colab_setup.ipynb — starts Ollama on Colab Pro, creates ngrok tunnel to your Windows machine - .env with COLAB_OLLAMA_URL=https://xxxx.ngrok-free.app - The ngrok URL changes every Colab session — update .env each time

    agentstatelib/core/state.py

    Import BaseModel and Field from pydantic. Import Literal and Any from typing. Import uuid and time.

    Write WorkflowStatus as a type alias using Literal with exactly four string values: "running", "complete", "failed", "paused". This is the complete set of valid workflow statuses — no other values are accepted.

    Write the Task model inheriting from BaseModel. Fields in order: - id typed as str with default_factory=lambda: str(uuid.uuid4()) - goal_id typed as str | None defaulting to None — tasks with no goal_id are ungrouped and not subject to goal-based invariant checks - description typed as str - status typed as Literal["pending", "running", "done", "failed"] defaulting to "pending" - result typed as Any | None defaulting to None - created_at typed as float with default_factory=time.time - updated_at typed as float with default_factory=time.time

    Docstring for Task: "A unit of work within a workflow. May optionally belong to a Goal via goal_id. When goal_id is None the task is ungrouped and not checked by goal-based invariants. The result field stores agent output after the task completes."

    Write the Goal model. Fields: - id typed as str, uuid factory - description typed as str - status typed as Literal["pending", "active", "complete", "failed"] defaulting to "pending" - created_at typed as float, time factory

    Docstring for Goal: "A high-level objective that groups related Tasks. Goals are optional — workflows that use only facts and artifacts do not need them."

    Write the Artifact model. Fields: - id typed as str, uuid factory - produced_by typed as str — the agent_id that created this artifact - artifact_type typed as str — free-form, examples: draft, source, summary, decision, report - content typed as Any - created_at typed as float, time factory

    Docstring for Artifact: "An output produced by an agent and stored in shared state. artifact_type is a free-form string — use whatever names make sense for your workflow domain, such as 'draft', 'source', or 'summary'."

    Write the Decision model. Fields: - id typed as str, uuid factory - made_by typed as str - description typed as str - rationale typed as str — the reasoning behind the decision, required for audit - timestamp typed as float, time factory

    Docstring for Decision: "A recorded workflow decision with rationale. Append to SharedState.decisions when an agent makes a significant choice that downstream agents or human reviewers should be able to audit."

    Write the SharedState model. Fields in order: - workflow_id typed as str, uuid factory - workflow_type typed as str defaulting to "general" - goal typed as str - goals typed as dict[str, Goal] with default_factory=dict - tasks typed as dict[str, Task] with default_factory=dict - artifacts typed as dict[str, Artifact] with default_factory=dict - decisions typed as list[Decision] with default_factory=list - facts typed as dict[str, Any] with default_factory=dict - status typed as WorkflowStatus defaulting to "running" - created_at typed as float, time factory - updated_at typed as float, time factory

    Docstring for SharedState: "The single source of truth for a workflow. Never mutated directly — all changes go through StatePatch and apply_patch which return a new SharedState object. All agents in a parallel round read the same SharedState snapshot, guaranteeing consistent reads within a round with no intra-round races. The event log stores the sequence of patches applied to state, so any past state can be reconstructed by replaying the log."

    agentstatelib/core/events.py

    Import BaseModel, Field, TypeAdapter from pydantic. Import Literal, Annotated, Union, Any from typing. Import uuid and time.

    Write BaseStateEvent inheriting from BaseModel. Fields: - event_id typed as str, uuid factory - workflow_id typed as str - agent_id typed as str - timestamp typed as float, time factory

    Docstring for BaseStateEvent: "Base class for all events in the agentstatelib event log. Every state change, conflict, checkpoint, and error produces a typed event appended to the store. Events are immutable once written."

    Write six event subclasses, each inheriting from BaseStateEvent. For each one, add a type field typed as a Literal with a single string value and that same string as the default. This discriminator field is what allows Pydantic to deserialize any serialized event back to the correct subclass.

    WorkflowStarted — type: Literal["workflow_started"] = "workflow_started". Extra fields: workflow_type typed as str, goal typed as str. Docstring: "Emitted at the start of AgentGraph.run(). Marks the beginning of the event sequence and carries the workflow metadata needed to reconstruct SharedState from scratch during replay."

    WorkflowCompleted — type: Literal["workflow_completed"] = "workflow_completed". Extra field: final_status typed as str. Docstring: "Emitted when AgentGraph.run() finishes normally. final_status is the workflow status at the time of completion."

    PatchApplied — type: Literal["patch_applied"] = "patch_applied". Extra fields: patch_id typed as str, target typed as str, old_value typed as Any, new_value typed as Any, reason typed as str. Note that timestamp is inherited from BaseStateEvent but here it carries the patch's own timestamp — the PatchApplied event's timestamp is the event creation time, not the patch creation time, so you may need to store the patch timestamp separately if needed. Docstring: "Emitted after a winning patch is applied to state. old_value and new_value store only the value at patch.target — not full state dumps. This keeps individual events compact regardless of overall state size."

    ConflictDetected — type: Literal["conflict_detected"] = "conflict_detected". Extra fields: conflict_id typed as str, path typed as str, winner_agent_id typed as str, loser_agent_id typed as str, resolution_strategy typed as str, existing_patch typed as dict[str, object] | None defaulting to None, incoming_patch typed as dict[str, object] | None defaulting to None. Docstring: "Emitted when resolve_batch() detects two patches targeting the same state path in the same parallel round. Both patches stored as plain dicts rather than StatePatch objects to guarantee JSON serializability regardless of what value fields contain."

    CheckpointSaved — type: Literal["checkpoint_saved"] = "checkpoint_saved". Extra fields: checkpoint_id typed as str, event_count typed as int. Docstring: "Emitted when save_checkpoint() completes successfully."

    AgentErrored — type: Literal["agent_errored"] = "agent_errored". Extra fields: error_type typed as str, error_message typed as str, retry_count typed as int. Docstring: "Emitted when an agent raises an unhandled exception. retry_count records how many retries were attempted."

    After the six subclasses, write StateEvent as a type alias. Use Annotated wrapping a Union of all six subclass types, with Field(discriminator="type") as the second argument to Annotated. This is the discriminated union — Pydantic uses the type field value to pick the correct subclass during deserialization.

    Write at module level: event_adapter = TypeAdapter(StateEvent). This is the object used for deserialization — callers call event_adapter.validate_json(raw_json_string) to get back the correct typed event subclass.

    agentstatelib/memory/store.py

    Import Protocol and runtime_checkable from typing. Import aiosqlite. Import json. Import StateEvent and event_adapter from agentstatelib.core.events. Import BaseStateEvent too.

    Write the StateStore protocol. Decorate it with @runtime_checkable. It should have four async methods: append(self, event: StateEvent) -> None, get_workflow(self, workflow_id: str) -> list[StateEvent], since(self, workflow_id: str, index: int) -> list[StateEvent], count(self, workflow_id: str) -> int. Each method body is just .... Docstring for StateStore: "Protocol for agentstatelib persistence backends. Any class implementing these four async methods satisfies this protocol and works as a drop-in backend for AgentGraph. No base class import required — structural typing means any conforming class qualifies automatically. Built-in implementations: InMemoryStore (for testing), SQLiteStore (for single-process use), PostgreSQLStore (for production)."

    Write InMemoryStore. It has one instance variable _events: dict[str, list[StateEvent]] initialized to an empty dict in __init__. Implement all four methods: - append adds the event to the list for that workflow_id, creating the list if the key does not yet exist - get_workflow returns a copy of the list (copy is critical — callers must not mutate the store's internal state), or an empty list if the workflow_id is not found - since returns the list sliced from the given index onward, or an empty list if the workflow_id is not found - count returns the length of the list, or 0 if the workflow_id is not found

    Docstring for InMemoryStore: "In-memory StateStore. Data is lost when the process exits. Use for tests and rapid development. Satisfies the StateStore protocol and passes the same test suite as SQLiteStore."

    Write SQLiteStore. It takes a path: str parameter in __init__, stored as self.path. Write a private async method _init_db(self, db) that executes CREATE TABLE IF NOT EXISTS events with columns: id INTEGER PRIMARY KEY AUTOINCREMENT, event_id TEXT NOT NULL, workflow_id TEXT NOT NULL, type TEXT NOT NULL, data TEXT NOT NULL, timestamp REAL NOT NULL. Then executes CREATE INDEX IF NOT EXISTS idx_wf_id ON events(workflow_id). Then calls await db.commit().

    Implement append: open a connection with async with aiosqlite.connect(self.path) as db, call await self._init_db(db), execute an INSERT with the five non-id fields, then await db.commit(). Serialize the event to JSON using event.model_dump_json(). The INSERT statement inserts event_id, workflow_id, event.type, the serialized JSON string, and event.timestamp.

    Implement get_workflow: open connection with async with aiosqlite.connect(self.path) as db, call await self._init_db(db), execute SELECT data FROM events WHERE workflow_id=? ORDER BY id ASC with the workflow_id as the parameter, call await cursor.fetchall(), return a list where each row's data column (row[0]) is deserialized with event_adapter.validate_json(row[0]).

    Implement since: same structure as get_workflow but the query adds LIMIT -1 OFFSET ? with the index as the second parameter. LIMIT -1 in SQLite means no limit on results, and OFFSET ? skips the first index rows. Return the deserialized list.

    Implement count: open connection, init db, execute SELECT COUNT(*) FROM events WHERE workflow_id=?, call await cursor.fetchone(), return row[0] as an integer.

    Docstring for SQLiteStore: "SQLite-backed StateStore. Events persist across process restarts. Each write is wrapped in a transaction. Suitable for single-process workflows and development. For production multi-process deployments, use PostgreSQLStore (requires asyncpg)."

    docs/concepts/conflict.md

    Write the conflict detection concept guide.

    Section 1 — What a conflict is. Two agents in the same parallel round both returning patches targeting the same dotted state path. Contrast with LangGraph: in LangGraph parallel node updates are merged with dict.update() which silently overwrites — the second writer wins and there is no record of the collision. In agentstatelib the same situation produces a ConflictDetected event, the configured resolver picks the winner, and both patches are preserved in the event log.

    Section 2 — When conflicts happen. Only within a round. If agent_a updates "facts.status" in round 1 and agent_b updates "facts.status" in round 2, that is not a conflict — agent_b is making a deliberate update to already-applied state. Conflicts are detected in resolve_batch() which processes all patches from a single round together.

    Section 3 — The resolution pipeline. Step by step: patches arrive in resolve_batch() → path collision detected → resolver.resolve(existing, incoming) called → winner determined → ConflictDetected event emitted → winning patch goes to apply_patch() → PatchApplied event emitted.

    Section 4 — Resolution strategies. Explain LastWriteWins (default, newer timestamp wins, ties go to incoming), PriorityBased (set the priority field on StatePatch, higher value wins, ties fall back to last-write-wins), RejectIncoming (first patch to a path wins, incoming is logged but not applied — useful when you want the first agent to lock a path for a round). When to use each.

    Section 5 — Custom resolver. Explain the ConflictResolver protocol: one method resolve(existing, incoming) -> StatePatch. Show a five-line pattern for writing a custom resolver. Note that any class with this method satisfies the protocol without importing anything from agentstatelib.

    Section 6 — Reading the conflict log. Show how to retrieve events from the store, filter for ConflictDetected instances, and read path, winner_agent_id, loser_agent_id, resolution_strategy.

    docs/concepts/parallelism.md

    Write the parallel execution concept guide.

    Section 1 — The round model. Every call to graph.run() executes in rounds. In each round: all agents whose incoming edges fire are collected, all run concurrently against the same state snapshot, patches are resolved, winning patches are applied, invariants are checked, the next round is computed. No agent in a round sees any other agent's updates until the next round.

    Section 2 — Why this matters. Eliminates read-write races within a round. All agents see identical state regardless of execution order. Conflicts are explicit and logged, not silent.

    Section 3 — Fan-out topology. coordinator→[analyst_a, analyst_b]. Both analysts run in round 2 simultaneously. Show the two edge declarations that produce this. Explain that the coordinator's state update is visible to both analysts since it was applied at the end of round 1 before round 2 begins.

    Section 4 — Fan-in. [analyst_a, analyst_b]→aggregator. Aggregator runs in round 3 after both analysts complete. Explain how _next_round deduplicates — the aggregator appears in the next round's list only once even though two edges point to it.

    Section 5 — Parallel start. run(state, start=["agent_a", "agent_b"]) runs multiple agents in round 1 simultaneously.

    Section 6 — The semaphore. max_concurrent in AgentGraph.__init__ controls an asyncio.Semaphore that limits total simultaneous agent coroutines across all rounds. Default is 10. Set it lower if your agents make API calls and you need to stay within rate limits.
