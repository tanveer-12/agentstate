    Store API

    StateStore

    Bases: Protocol

    Protocol for agentstatelib persistence backends.

    Any class implementing these five async methods satisfies this protocol and works as a drop-in backend for AgentGraph.

    No base class import required — structural typing means any conforming class qualifies automatically.

    Built-in implementations: - InMemoryStore (for testing) - SQLiteStore (for single-process use) - PostgreSQLStore (for production)

    append(event) async

    count(workflow_id) async

    get_workflow(workflow_id) async

    list_workflows() async

    Return all unique workflow_ids in the store, most recent first.

    since(workflow_id, index) async

    InMemoryStore

    In-memory StateStore.

    Data is lost when the process exits. Use for tests and rapid development.

    Satisfies the StateStore protocol and passes the same test suite as SQLiteStore.

    _events = {} instance-attribute

    __init__()

    append(event) async

    Append an event to the workflow event stream.

    count(workflow_id) async

    Return event count for a workflow.

    get_workflow(workflow_id) async

    Return all events for a workflow.

    list_workflows() async

    Return all unique workflow_ids in the store, most recent first.

    since(workflow_id, index) async

    Return events starting at the supplied index.

    SQLiteStore

    SQLite-backed StateStore.

    Events persist across process restarts. Each write is wrapped in a transaction.

    Suitable for single-process workflows and development.

    For production multi-process deployments, use PostgreSQLStore (requires asyncpg).

    PostgreSQLStore

    PostgreSQL-backed StateStore for production multi-process deployments.

    Requires asyncpg: pip install asyncpg. DSN format: postgresql://user:password@host/database. Connection pool is created lazily on first use. Call close() when done.

    Checkpoint

    Bases: BaseModel

    A snapshot of SharedState at a point in time. event_count marks where in the event log the workflow had progressed, so recovery can resume from the store without replaying the entire history from scratch.

    replay

    Reconstruct SharedState from an event log by finding the WorkflowStarted event for metadata then applying all PatchApplied events in timestamp order.

    Pass a truncated list to reconstruct any past state.

    ReplayDebugger

    Step through a workflow's event log to inspect state at any point. When a workflow produces unexpected results, use this to find the exact moment state diverged.

    No equivalent tool exists in LangGraph, CrewAI, or AutoGen.
